# 03 首页侧栏与会话联动

对应总纲：27.2 的首页部分

## 目标

首页右侧侧栏不应该是固定模式的后台面板，而应该是跟随当前对话内容动态弹出、动态绑定的工作台。

它至少要理解两类状态源：

1. 创建态会话
2. 运行态会话

5. `peek` 态目前只是轻量入口，不是完整摘要卡。

## 代码仓判断

首页侧栏当前属于“动态联动已落地，但产品编排仍待收口”。

它已经不是空白页，但离文档要求还有明显距离：

1. 有壳
2. 有创建态和运行态双状态源
3. 有语义驱动切换
4. 缺更细粒度的信息分层与交互收口

## 已完成部分

1. 首页已经有统一容器承载 workflow / supervisor / agent 相关操作。
2. run 级别的 workflow 状态轮询已经具备。
3. 聊天会话和 workflow run 的绑定机制已经存在。
4. 已经具备从聊天触发结构化操作入口的基础。
5. 创建态结果已经能回流首页会话。
6. 侧栏已会根据当前对话内容动态展开，而不是固定模式常驻。

## 未完成部分

1. 创建态摘要卡和创建态历史的进一步强化
2. 从运行态回看创建态设计的更强对照视图
3. 对话意图编排器从规则式升级为更稳定的语义层
4. 首页卡片结果向侧栏 / sheet / modal 的系统性迁移
5. “手动绑定”心智的进一步削弱

## 改进项

1. 在首页侧栏状态模型里新增：
   - `creationSession`
   - `currentRunBinding`
   - `recentRunBindings`
2. 把侧栏视图明确拆成：
   - 创建态
   - 运行态
   - 历史摘要态
3. 把“查看 workflow / 启动 workflow / 查看汇报 / 创建 Agent”从固定 tab 变成意图驱动面板。
4. 用户在首页发起 workflow 创建后，立即把创建态写回当前 chat session。
5. workflow 启动后，侧栏自动从创建态切到当前 run 的 supervisor 视图。
6. 运行结束后允许从侧栏直接对比“创建态设计”和“运行态结果”。
7. 系统性梳理 AI card/action：
   - 无效按钮直接删
   - 可保留按钮改为打开侧栏 / sheet / modal
   - 少把操作继续“发回聊天”
8. 将首页右侧指挥区继续收口成稳定控制区，而不是零散卡片拼装。

## 优先级判断

首页侧栏是最容易被用户感知到完成度的区域，但它不应该先于底层状态乱做。

推荐顺序：

1. 先补创建态 session / `OpenSpec` store
2. 再补首页双态侧栏
3. 最后补语义驱动编排和细节动画
